## Artifact title

> Allow single-character OData identifiers

## Artifact body

## Bug
`cf-modkit-odata` rejects valid filters whose field identifier has one character, such as `x eq 1`.

## Cause
The PEG `identifier` rule in `libs/modkit-odata/src/odata_parse.rs:98-99` requires a valid first character plus one or more additional identifier characters. That makes the effective minimum identifier length two, so `x eq 1` fails at the space after `x`.

## Fix
Change only the suffix repetition in the `identifier` rule from `+` to `*`, keeping the first-character restriction unchanged. Retain the regression test in `libs/modkit-odata/src/odata_parse_tests.rs` for `parse_str("x eq 1")` producing an equality comparison against numeric value `1`.

## Verification

```sh
cargo test -p cf-modkit-odata --lib single_character_identifier
```

After the grammar edit, this focused test should pass; then run `cargo test -p cf-modkit-odata --lib` to check the parser library suite.

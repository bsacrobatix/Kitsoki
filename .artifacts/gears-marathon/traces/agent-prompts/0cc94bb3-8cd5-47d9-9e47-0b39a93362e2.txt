## Artifact title

> Fix Kubernetes-safe env overrides for dashed gear names

## Artifact body

## Bug
Kubernetes-safe env vars such as `APP__GEARS__STATIC_AUTHZ_PLUGIN__CONFIG__PRIORITY=50` do not override YAML gear entries keyed as `static-authz-plugin`; the gear keeps its YAML value of `100`.

## Cause
`AppConfig::load_layered` merges `Env::prefixed("APP__").split("__")` directly into `AppConfig.gears` in `libs/toolkit/src/bootstrap/config/mod.rs:321-325`. Figment maps the env var to `gears.static_authz_plugin.config.priority`, but the YAML map key is `static-authz-plugin`, so the merge misses the existing entry.

## Proposed Fix
Add a narrow post-extraction reconciliation for `AppConfig.gears`: for dashed gear names, map the Kubernetes-safe underscore alias back onto the dashed key, deep-merge the alias value into the dashed value with env precedence, and remove the alias entry. Do not rewrite or steal overrides from real underscore-named gears; if both dashed and underscore forms are explicit gear names, leave the underscore key behavior unchanged.

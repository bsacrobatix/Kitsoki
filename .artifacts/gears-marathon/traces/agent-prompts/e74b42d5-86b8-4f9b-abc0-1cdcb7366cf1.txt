You are implementing the fix for **bug1** — *bug1 (gh-4115) — Kubernetes env-var override of a dashed gear name is silently dropped. A gear named with dashes (e.g. `static-authz-plugin`) is stored in AppConfig.gears under the dashed key. In Kubernetes, env var names cannot contain dashes (C_IDENTIFIER rule), so operators MUST use underscores: APP__GEARS__STATIC_AUTHZ_PLUGIN__CONFIG__PRIORITY=50. The figment Env::prefixed("APP__").split("__") pipeline lowercases and produces key path gears.static_authz_plugin.config.priority, but the HashMap entry is keyed `static-authz-plugin` (dashes), so the lookup misses and the override is silently ignored — the gear keeps its YAML value (e.g. priority 100) instead of the overridden 50. Expected: the underscore env-var form should override the dashed gear. Source: libs/toolkit/src/bootstrap/config/mod.rs (AppConfig::load_layered). Tests: cargo test -p cf-gears-toolkit --features bootstrap --lib bootstrap::config. A gear whose name already uses underscores is the negative control and already works.*
inside the worktree at `/Users/brad/code/gears-rust/.worktrees/marathon-bug1-gpt`.

The proposing room produced this proposal:

> Add a small post-extraction reconciliation step in AppConfig::load_layered for gear keys only. After extracting AppConfig from Figment, inspect existing gear names that contain dashes and compute their Kubernetes-safe alias by replacing '-' with '_'. If config.gears contains that alias and the alias is not also an explicitly configured gear name, deep-merge the alias value into the dashed gear value with alias/env values taking precedence, then remove the alias entry. Keep underscore-named gears unchanged, and leave ambiguous cases where both foo-bar and foo_bar are explicit gear names untouched to avoid stealing overrides from the real underscore gear. Add or keep the RED test test_env_override_uses_kubernetes_safe_name_for_dashed_gear to assert both the dashed gear and underscore control resolve to priority 50.

**Root cause:** In libs/toolkit/src/bootstrap/config/mod.rs:321-325, AppConfig::load_layered merges Figment defaults, YAML, and Env::prefixed("APP__").split("__") directly into AppConfig.gears. Figment lowercases APP__GEARS__STATIC_AUTHZ_PLUGIN__CONFIG__PRIORITY into the path gears.static_authz_plugin.config.priority, but it does not translate underscores back to dashes. Because the YAML gear is stored under the HashMap key static-authz-plugin, the env layer updates or creates static_authz_plugin instead of merging into the existing dashed key, so the dashed gear retains its YAML value.

**Affected files:** libs/toolkit/src/bootstrap/config/mod.rs

> bug6 verified: OData parser rejects single-character identifiers

## Artifact body

## What is broken
`cf-modkit-odata` rejects a valid OData filter when the field identifier is a single character. The new regression test asserts that `parse_str("x eq 1")` succeeds and yields an equality comparison between identifier `x` and numeric value `1`.

## Reproduction
I added `odata_parse_tests::single_character_identifier` in `libs/modkit-odata/src/odata_parse_tests.rs` and ran the focused lib test command:

```sh
cargo test -p cf-modkit-odata --lib single_character_identifier
```

The test is RED on the current tree. It panics because `parse_str("x eq 1")` returns `ParseError::Parsing` at column 2 instead of an AST.

## Evidence
- Test file: `libs/modkit-odata/src/odata_parse_tests.rs`
- Failing command output includes: `single-character identifiers are valid: Parsing("error at 1:2: expected ['a'..='z'|'A'..='Z'|'_'|'0'..='9']")`

## Implicated services/components
- `cf-modkit-odata` parser implementation in `libs/modkit-odata/src/odata_parse.rs`, specifically the PEG `identifier` rule requiring one or more trailing identifier characters after the first character.
- `cf-modkit-odata` filter parse API exposed through `odata_filters::parse_str`, which surfaces the parse failure to callers.

## Decision criteria

- **accept** — `bug_verified` is true, evidence is concrete (test, script,
  recorded sequence), `involved_components` are plausible, and the
  reproduction can be re-run by another engineer from the steps alone.
- **refine** — the artifact is on the right track but missing something
  important (e.g. no executable repro, fuzzy expected/actual outcome,
  involved_components feel made-up). Put the specific gap in `reason`;
  the next iteration's prompt will see it as feedback.
- **restart_from** — the wrong bug is being reproduced; the engineer
  misread the ticket. Reset to idle / reproducing.
- **quit** — this bug is unreproducible or out-of-scope (e.g. user error,
  duplicate of a closed ticket). Rare.
- **uncertain** — you cannot judge from the artifact alone (e.g. it
  refers to a service you have no context on). Yield to a human.

Set `confidence` honestly. Confidence below the project threshold (default
0.8) means a human will be asked even in `llm_then_human` mode; in strict
`llm` mode an uncertain verdict leaves the state held until an operator
intervenes.

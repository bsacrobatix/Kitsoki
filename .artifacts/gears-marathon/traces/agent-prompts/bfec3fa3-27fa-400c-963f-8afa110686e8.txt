# Reviewing tests & implementation — produce the testing artifact

You are reviewing the implementation of the fix for **bug1** —
*bug1 (gh-4115) — Kubernetes env-var override of a dashed gear name is silently dropped. A gear named with dashes (e.g. `static-authz-plugin`) is stored in AppConfig.gears under the dashed key. In Kubernetes, env var names cannot contain dashes (C_IDENTIFIER rule), so operators MUST use underscores: APP__GEARS__STATIC_AUTHZ_PLUGIN__CONFIG__PRIORITY=50. The figment Env::prefixed("APP__").split("__") pipeline lowercases and produces key path gears.static_authz_plugin.config.priority, but the HashMap entry is keyed `static-authz-plugin` (dashes), so the lookup misses and the override is silently ignored — the gear keeps its YAML value (e.g. priority 100) instead of the overridden 50. Expected: the underscore env-var form should override the dashed gear. Source: libs/toolkit/src/bootstrap/config/mod.rs (AppConfig::load_layered). Tests: cargo test -p cf-gears-toolkit --features bootstrap --lib bootstrap::config. A gear whose name already uses underscores is the negative control and already works.*.

The proposed fix was:

> ## Bug
Kubernetes-safe env vars such as `APP__GEARS__STATIC_AUTHZ_PLUGIN__CONFIG__PRIORITY=50` do not override YAML gear entries keyed as `static-authz-plugin`; the gear keeps its YAML value of `100`.

## Cause
`AppConfig::load_layered` merges `Env::prefixed("APP__").split("__")` directly into `AppConfig.gears` in `libs/toolkit/src/bootstrap/config/mod.rs:321-325`. Figment maps the env var to `gears.static_authz_plugin.config.priority`, but the YAML map key is `static-authz-plugin`, so the merge misses the existing entry.

## Proposed Fix
Add a narrow post-extraction reconciliation for `AppConfig.gears`: for dashed gear names, map the Kubernetes-safe underscore alias back onto the dashed key, deep-merge the alias value into the dashed value with env precedence, and remove the alias entry. Do not rewrite or steal overrides from real underscore-named gears; if both dashed and underscore forms are explicit gear names, leave the underscore key behavior unchanged.

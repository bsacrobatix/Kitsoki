> bug1 gh-4115 Kubernetes-safe env overrides for dashed gear names

## Artifact body

bug1 was caused by a key-shape mismatch in layered config loading. A dashed gear such as `static-authz-plugin` is stored in `AppConfig.gears` under its dashed name, while Kubernetes-safe environment variables must use underscores. Figment's `APP__...` env pipeline produced an override path like `gears.static_authz_plugin.config.priority`, so the override landed under an underscore alias and never reached the dashed gear entry.

The fix reconciles Kubernetes-safe aliases after layered extraction and gear-file merging in `libs/toolkit/src/bootstrap/config/mod.rs`. For dashed gear names, underscore alias values are deep-merged into the real dashed gear entry. The reconciliation intentionally skips alias folding when an explicit underscore-named gear exists, preserving that gear as a separate configuration target.

Validation covered the original failure mode with `APP__GEARS__STATIC_AUTHZ_PLUGIN__CONFIG__PRIORITY=50`, verifying the dashed `static-authz-plugin` gear receives priority `50` instead of keeping the YAML value. The underscore-named control gear remains valid. `cargo check -p cf-gears-toolkit --features bootstrap` passed, and `cargo test -p cf-gears-toolkit --features bootstrap --lib bootstrap::config` passed with 63 passed, 0 failed, 0 ignored, and 252 filtered out.

No refinement cycles were needed. The only misleading signal was an earlier Go-style `./...` command failure, which was not applicable to this Cargo workspace. For similar bugs, the useful approach is to validate the exact runtime key translation path and include a negative-control name collision case, not just the successful alias mapping.

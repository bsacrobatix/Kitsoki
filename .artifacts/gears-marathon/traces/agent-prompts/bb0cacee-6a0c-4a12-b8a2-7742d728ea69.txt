*bug4 — CanonicalError cannot be produced from common library errors. The `cf_modkit_errors::CanonicalError` type does not implement `From<std::io::Error>` or `From<serde_json::Error>`, so the `?` operator cannot propagate those errors into a `Result<_, CanonicalError>` and there is no ergonomic conversion. Source: libs/modkit-canonical-errors/src/error.rs (lib name cf_modkit_errors). Tests: cargo test -p cf-modkit-canonical-errors --test error.*.

The proposed fix was:

> ## Bug
`cf_modkit_errors::CanonicalError` cannot be created from `std::io::Error` or `serde_json::Error`, so both `CanonicalError::from(err)` and `?` propagation into `Result<_, CanonicalError>` fail to compile.

## Cause
`libs/modkit-canonical-errors/src/error.rs` implements `Display` and `std::error::Error` for `CanonicalError`, but it does not implement `From<std::io::Error>` or `From<serde_json::Error>`. Rust therefore has no conversion path for direct `from` calls or for the implicit conversion used by `?`.

## Fix
Add two local `From` impls in `libs/modkit-canonical-errors/src/error.rs`:

- `std::io::Error` -> `CanonicalError::internal(err.to_string()).create()` for 500 / `Internal`.
- `serde_json::Error` -> invalid-argument canonical error using `InvalidArgument::format(err.to_string())`, preserving the serde message as `detail()`, for 400 / `Invalid Argument`.

No dependency changes are required; `serde_json` is already in the crate dependencies.

## Confidence
